## 一、技術選型

| 層次 | 選擇 | 理由 |
|------|------|------|
| 框架 | Next.js 15 App Router | SSR + 快速路由切換，`layout.tsx` 實現巢狀 UI |
| UI 元件 | shadcn/ui + Tailwind CSS | 符合 `01-專案目標.md` 規範；無頭元件易於客製 |
| 伺服器狀態 | TanStack Query v5 | API 快取、背景輪詢（索引狀態、逐字稿）、樂觀更新 |
| 表單驗證 | React Hook Form + Zod | 型別安全，與後端 schema 共用驗證規則 |
| 圖示 | Lucide React | shadcn/ui 預設使用，風格一致（**⚠️ 設計備注**：避免過度 AI 感的圖示；基於 Vexa 授權條款，不得在 UI 中展示 Vexa 商標或 Logo） |
| 檔案上傳 | react-dropzone | 拖拉、點擊、剪貼簿上傳 |
| 通知 | sonner | shadcn/ui 整合的 Toast |

### 認證 Token 策略（與 04-API設計.md 補充）

```
useSession() → session.vexaToken（Vexa API token）
  ↓
呼叫 App Backend 時：Authorization: Bearer <vexaToken>
  ↓
Backend 驗證：查詢 public.api_tokens WHERE token = vexaToken → 取得 user_id
```

這比 NextAuth JWT 更簡單：`vexaToken` 已在 `session` 中，無需額外處理。

## 三、路由與頁面責任

### 3.1 `/projects` — 專案總覽

**責任**：顯示使用者參與的所有專案、提供建立新專案入口。Sidebar 提供全局「建立會議」與 Meetings 入口。

```
┌─ Sidebar ─────────┬─ Main Content ────────────────────────────────┐
│                   │                                               │
│  🤖 MeetBot       │  我的專案              [+ 建立專案]            │
│                   │  🔍 搜尋...  [全部▼]  [倒序▼]                 │
│  [+ 建立會議]     │                                               │
│                   │  ┌───────────────────────────────────────────┐│
│  📁 Projects      │  │ 🏢 Q3 產品規劃  Owner  3成員  7份資料      ││
│  🗂 Meetings      │  │    1個進行中會議                  [進入 →] ││
│                   │  ├───────────────────────────────────────────┤│
│  ── 最近專案 ──   │  │ 📁 行銷企劃    成員  5成員  12份資料       ││
│  ▶ Q3 產品規劃   │  │    無進行中會議                  [進入 →] ││
│    行銷企劃       │  └───────────────────────────────────────────┘│
│                   │                                               │
│  ─────────────    │                                               │
│  👤 user@..       │                                               │
└───────────────────┴───────────────────────────────────────────────┘
```

> **Sidebar 說明**：
> - **`[+ 建立會議]`**：全局入口，開啟 `GlobalCreateMeetingDialog`（含關聯專案選擇）
> - **`📁 Projects`**：切換至 `/projects` 頁面
> - **`🗂 Meetings`**：切換至 `/meetings` 頁面（全局會議列表）
> - **左下角 `📬 信箱`（`inbox-button.tsx`）**：顯示寄給我的待處理邀請數（`useMyInvitations`，每 60 秒輪詢），
>   下拉可直接接受/拒絕；接受後 invalidate `['projects']` 讓新專案立即出現
> - 搜尋欄 + `[全部▼]`（類型篩選）+ `[倒序▼]`（排序）在主內容區

**元件樹**
```
ProjectsPage
├── ProjectFilterBar         ← search、type、order 篩選
└── ProjectCard[]            ← GET /projects（帶篩選參數）
    └── CreateProjectDialog (modal)

Sidebar
└── GlobalCreateMeetingDialog  ← POST /meetings（projectId 選填）
    └── ProjectSelector         ← GET /projects（篩選具 canMeeting 的專案）
```

**摘要生成中（ENDED 但 summary 為 null）：**
```
│  📋 摘要                                                  │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  ⏳ 蜜塔正在生成會議摘要...（通常需要 10-30 秒）      │ │

```

**失敗（FAILED）：可重邀，非終止態**
```
┌───────────────────────────────────────────────────────────┐
│ ← 返回   每週同步   ❌ 加入失敗        [🔄 重新邀請蜜塔]     │
├───────────────────────────────────────────────────────────┤
│  ⚠️ 蜜塔加入失敗。常見原因與處理方式：                       │
│   • 候客室未放行：請開啟「快速存取」或手動允許蜜塔進入         │
│   • 會議限制訪客：改用允許訪客的會議                          │
│   • 服務暫時性問題：可直接重試                                │

```
> **FAILED 不再是終止態**：右上角「🔄 重新邀請蜜塔」呼叫 `POST .../bot/reinvite`，
> 把會議轉回 PENDING 重試，**不需重建 MeetingInstance**。ENDED 會議同樣顯示「重新邀請」可再次加入。
> PENDING（加入中）則顯示「取消」按鈕（`cancel-meeting-button` → `POST .../cancel`）。

**右上角操作按鈕（依 status，且需 `canMeeting`）**：

|--------|------|------|

// 取得專案內會議清單

function useAllMeetings(params?: {
  search?: string
  since?: 1 | 3 | 7
  order?: 'asc' | 'desc'
  status?: MeetingStatus
}): UseQueryResult<PaginatedMeetings>  // MeetingRow 含 projectId / projectName（可為 null）

// 取得單一會議（含摘要生成輪詢）
// projectId 為 null 時使用全局端點 GET /meetings/:id（無關聯專案的會議）
function useMeeting(projectId: string | null, meetingId: string) {
  return useQuery({
    queryKey: ['meeting', meetingId],
    queryFn: () => projectId
      ? api.getMeeting(projectId, meetingId)
      : apiClient.get<MeetingDetail>(`/meetings/${meetingId}`),
    refetchInterval: (query) => {
      const m = query.state.data
      // Bot 加入中：繼續輪詢
      if (m?.status === 'PENDING') return 3000
      // 會議進行中：繼續輪詢以偵測 ACTIVE → ENDED 狀態轉換
      // （Bot 自動離會後才知道 status 變 ENDED，父元件依此切換 isActive → false，停止逐字稿輪詢）
      if (m?.status === 'ACTIVE') return 5000
      // 摘要生成中：繼續輪詢直到 summary 不為 null
      // ⚠️ 條件必須用 === null，不可用 !m?.summary：
      //   後端在「無逐字稿」或「摘要生成失敗」時設 summary = ''（空字串作為 sentinel）。
      //   !'' 為 true，若用 !m?.summary 則空字串同樣觸發輪詢，輪詢永遠不停止。
      //   null  → 摘要尚未嘗試生成（繼續輪詢）
      //   ''    → 已嘗試但無內容（停止輪詢，UI 顯示「無摘要」）
      //   string → 有內容（停止輪詢）
      if (m?.status === 'ENDED' && (m?.summary === null || m?.summary === undefined)) return 5000
      // ENDED（summary 非 null，含空字串）或 FAILED：終止態，停止輪詢
      // FAILED 由 MeetingFailedBanner 處理，顯示靜態錯誤提示，無重試按鈕
      return false
    }
  })
}

// 進行中會議的即時逐字稿輪詢
// ⚠️ ACTIVE 會議的 API 使用 since_start_time（float）游標，非 since_id（integer）
// 04-API設計.md：GET .../transcriptions?since_start_time=X 回傳 start_time >= X 的 segments
// projectId 為 null 時使用全局端點 GET /meetings/:id/transcriptions（無關聯專案的會議）
//
// 去重說明：後端 filter 為 >=，cursor 邊界 segment 會被重複回傳；網路重試也可能帶重複。
// 使用 Map<segmentKey, segment> upsert 去重：
//   優先用 segmentId（Vexa segment_id，穩定唯一）；為 null 時退而用 `${startTime}-${text}`
// Map upsert 同時處理 Vexa mutable segment 更新（相同 ID 覆蓋舊值）。
function useLiveTranscriptions(
  projectId: string | null,
  meetingId: string,
  isActive: boolean   // 只有 ACTIVE 才輪詢
) {
  const [sinceStartTime, setSinceStartTime] = useState<number | null>(null)
  const [segmentMap, setSegmentMap] = useState<Map<string, TranscriptSegment>>(new Map())

  useQuery({
    queryKey: ['transcriptions-live', meetingId],
    queryFn: async () => {
      const basePath = projectId
        ? `/projects/${projectId}/meetings/${meetingId}/transcriptions`
        : `/meetings/${meetingId}/transcriptions`
      const url = sinceStartTime !== null
        ? `${basePath}?since_start_time=${sinceStartTime}`
        : basePath
      const res = await apiClient.get<TranscriptResponse>(url)
      if (res.items.length > 0) {
        setSegmentMap(prev => {
          const next = new Map(prev)
          for (const seg of res.items) {
            const key = seg.segmentId ?? `${seg.startTime}-${seg.text}`
            next.set(key, seg)
          }
          return next
        })
        setSinceStartTime(res.items.at(-1)!.startTime)  // float（Vexa 的 start_time）
      }
      return res
    },
    enabled: isActive,
    refetchInterval: 3000,
  })

  // 依 startTime 排序後輸出，保持逐字稿顯示順序一致
  return useMemo(
    () => [...segmentMap.values()].sort((a, b) => a.startTime - b.startTime),
    [segmentMap],
  )
}

// 會議 Bot 操作（mutation；projectId 為 null 時打全局端點）
function useBotLeave(projectId: string | null, meetingId: string): UseMutationResult     // POST .../bot/leave
function useCancelMeeting(projectId: string | null, meetingId: string): UseMutationResult // POST .../cancel（PENDING）
function useBotReinvite(projectId: string | null, meetingId: string): UseMutationResult   // POST .../bot/reinvite（FAILED/ENDED）

// ── 權限 ────────────────────────────────────────────

// 從 project 資料推導當前使用者的操作權限

  const { data: project } = useProject(projectId)
  return project?.permissions ?? {
    canView: false, canEdit: false,
    canDelete: false, canManage: false, canMeeting: false
  }
}

## 五、API Client 實作

```typescript
// lib/api-client.ts
import { getSession } from 'next-auth/react'

const BASE_URL = process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:4000'

async function getAuthHeader(): Promise<HeadersInit> {
  const session = await getSession()
  // session.vexaToken 由 NextAuth session callback 注入（Vexa 原本即有）
  const token = (session as any)?.vexaToken
  if (!token) throw new Error('Not authenticated')
  return { Authorization: `Bearer ${token}` }
}

export const apiClient = {
  async get<T>(path: string): Promise<T> {
    const headers = await getAuthHeader()
    const res = await fetch(`${BASE_URL}${path}`, { headers })
    if (!res.ok) throw await res.json()
    return res.json()
  },

  async post<T>(path: string, body?: unknown): Promise<T> {
    const headers = await getAuthHeader()
    const res = await fetch(`${BASE_URL}${path}`, {
      method: 'POST',
      headers: { ...headers, 'Content-Type': 'application/json' },
      body: body ? JSON.stringify(body) : undefined,
    })
    if (!res.ok) throw await res.json()
    return res.json()
  },

  async postForm<T>(path: string, formData: FormData): Promise<T> {
    const headers = await getAuthHeader()  // 注意：multipart 不設 Content-Type，讓瀏覽器自動設
    const res = await fetch(`${BASE_URL}${path}`, {
      method: 'POST',
      headers,  // 不包含 Content-Type
      body: formData,
    })
    if (!res.ok) throw await res.json()
    return res.json()
  },

  async patch<T>(path: string, body: unknown): Promise<T> {
    const headers = await getAuthHeader()
    const res = await fetch(`${BASE_URL}${path}`, {
      method: 'PATCH',
      headers: { ...headers, 'Content-Type': 'application/json' },
      body: JSON.stringify(body),
    })
    if (!res.ok) throw await res.json()
    return res.json()
  },

  async delete(path: string): Promise<void> {
    const headers = await getAuthHeader()
    const res = await fetch(`${BASE_URL}${path}`, { method: 'DELETE', headers })
    if (!res.ok) throw await res.json()
  },
}
```

## 十、輪詢策略總覽

| 場景 | Hook | 頻率 | 停止條件 |
|------|------|------|---------|
| 檔案索引狀態 | `useMaterialStatus` | 5 秒 | status ∉ {PENDING, PROCESSING} |
| Bot 加入中 | `useMeeting` | 3 秒 | status ≠ PENDING |
| 會議進行中（偵測結束） | `useMeeting` | 5 秒 | status ≠ ACTIVE |
| 即時逐字稿 | `useLiveTranscriptions` | 3 秒 | 父元件傳入 `isActive=false`（由 `useMeeting` 回傳 ENDED 觸發）；Map 去重，cursor 用 >= 過濾 |
| 摘要生成中 | `useMeeting` | 5 秒 | summary 不為 null（含空字串 ''；後端在無逐字稿或失敗時設 ''） |
