## 三、步驟

### 3.1 啟動 vexa-lite

執行 `docs/09-實作計畫/00-vexa-lite-local.md` 中的指令（含憑證，已加入 `.gitignore`）：

1. 執行 **§1 docker run 指令**，等待容器啟動（約 1–2 分鐘）
2. 執行 **§2 init-db**，初始化 Supabase 的 `public` schema
3. 執行 **§3 確認啟動**，確認容器狀態為 `Up`

> vexa-lite 啟動並執行 init-db 後，Supabase 的 `public` schema 會建立
> `users`、`meetings`、`transcriptions` 等表。

### 3.2 Supabase 初始設定

**建立 `app` schema**（在 Supabase SQL Editor 執行）：

```sql
CREATE SCHEMA IF NOT EXISTS app;
GRANT ALL ON SCHEMA app TO postgres;
GRANT USAGE ON SCHEMA app TO authenticated;
```

**建立 Storage Bucket**（在 Supabase Dashboard → Storage → New Bucket）：

| 設定項 | 值 |
|-------|-----|
| Bucket name | `meeting-materials` |
| Public | 否（Private） |
| File size limit | `15` MB |
| Allowed MIME types | `application/pdf, application/vnd.openxmlformats-officedocument.wordprocessingml.document, text/plain, text/markdown` |

### 3.3 Dify 設定

1. **匯入 Q&A Chatflow**：Dify 平台 → Studio → Import DSL → 選擇 `docs/01-edu2.yml`
2. **設定 Chatflow 的 API Key**（擇一）：
   - 在 Dify 平台 Settings → Environment Variables 新增 `DIFY_DATASET_API_KEY = dataset-...`
   - 或直接在 HTTP Request node 的 headers 欄位填入實際 Key
3. **建立會議摘要 Workflow**：依 `docs/01-RAG_API_串接文件_v1.1.md` 說明建立
4. **取得四把 API Key**：

| 變數名 | 格式前綴 | 對應用途 |
|--------|---------|---------|
| `DIFY_DATASET_API_KEY` | `dataset-` | Knowledge Base 操作 |
| `DIFY_WORKFLOW_API_KEY` | `app-` | RAG Q&A Chatflow |
| `DIFY_SUMMARY_WORKFLOW_API_KEY` | `app-` | 檔案摘要 Workflow |
| `DIFY_MEETING_SUMMARY_WORKFLOW_API_KEY` | `app-` | 會議摘要 Workflow |

### 3.4 後端 .env 設定

```bash
cd backend
cp .env.example .env
```

填入 `.env`（所有欄位均必填，見 `docs/03-資料庫Schema設計.md §六` 與 `docs/04-API設計.md §十三`）：

```bash
# Supabase
DATABASE_URL="postgresql://postgres:[PWD]@db.[PROJ].supabase.co:5432/postgres?schema=app"
SUPABASE_URL="https://[PROJ].supabase.co"
SUPABASE_SERVICE_ROLE_KEY="..."
SUPABASE_STORAGE_BUCKET="meeting-materials"

# Dify
DIFY_API_BASE="https://api.dify.ai/v1"
DIFY_DATASET_API_KEY="dataset-..."
DIFY_WORKFLOW_API_KEY="app-..."
DIFY_SUMMARY_WORKFLOW_API_KEY="app-..."
DIFY_MEETING_SUMMARY_WORKFLOW_API_KEY="app-..."
DIFY_CHATFLOW_TIMEOUT_MS=45000

# Anthropic
ANTHROPIC_API_KEY="sk-ant-..."

# Vexa WebSocket
VEXA_WS_URL="ws://localhost:8056"   # vexa-lite 的 WebSocket 端點

# Server
APP_PORT=4000
APP_CORS_ORIGINS="http://localhost:3000"
```

### 3.5 Prisma 初始化（需 vexa-lite 運行中）

```bash
cd backend

# 1. 同步 Vexa 的 public schema 定義
npx prisma db pull
# ⚠️ 執行後務必 diff，只保留 User / Meeting / Transcription 三個 model
# 其他 Vexa model 一律刪除，確認 app schema models 未被覆蓋

# 2. 建立 app schema 的初始 migration
npx prisma migrate dev --name init_app_schema
```

完成所有步驟後，確認以下項目後才開始 Phase 1：

- [ ] Supabase Storage 存在 `meeting-materials` bucket
- [ ] `npx prisma migrate status` 顯示 migration 已套用
- [ ] Dify Q&A Chatflow 已匯入，且 `DIFY_DATASET_API_KEY` 已設定
- [ ] `.env` 所有欄位均已填入實際值（無空欄位）
