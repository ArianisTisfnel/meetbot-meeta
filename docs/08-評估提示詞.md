① 前提是否明確：你的判斷是否依賴某個文件中未明確說明的架構
  假設（例如某個 API 的呼叫方式或資料來源）？若是，請在問題
  描述中列出該假設，讓我能快速確認或否定，再判定是否為真正
  的問題。

② 設計選擇還是遺漏：這是一個可辯護的 trade-off（只需提醒），
  還是若不修正實作時必然出錯的遺漏？只有後者才算問題；前者
  請標記「建議」而非「問題」。

③ 是否有具體佐證：每個問題需附上文件段落或 vexa/ 原始碼的
  檔案:行號作為依據。若只有邏輯推論而無直接佐證，請標注
  「待確認」並說明需要確認什麼。
